# 🍼 AguMi | AI Maternity Coach

An early-stage MVP of an AI-powered maternity coach designed to alleviate postpartum anxiety in the fourth trimester. 

🔗 **[Live Demo](https://agumi.poletta.cz)** *(Note: Link to your live project)*

## 🚀 The Problem It Solves
The **"Tracking Paradox"**: Many modern FemTech apps force exhausted mothers to maniacally track every minute of sleep and feeding, causing neurosis and obsessive states. 

**AguMi** takes a different approach. It offloads the cognitive burden using Smart Nudges and an intelligent symptom analyzer with a built-in **Red Flag Override** system. It supports the mother, validates her emotions, and strictly monitors critical medical markers without causing panic.

## ✨ Key Features
* **AI Symptom Checker:** Analyzes real-time input. If critical "Red Flags" (e.g., fever, heavy bleeding, projectile spit-up) are detected, the AI overrides standard responses and issues an immediate medical alert.
* **UX Tooltips (De-jargonization):** Medical terms (like *mastitis*, *diastasis recti*, *involution*) are automatically detected and explained via elegant tooltips on hover. No more googling scary symptoms in the middle of the night.
* **Smart Routine Journal:** Quick logging for feeding, sleep, and pumping without stressful timers.
* **Medical Library Search:** Simulated search across evidence-based databases (PubMed, Mayo Clinic) with bilingual support.
* **Midnight Premium UI:** A luxurious Dark Mode designed specifically for late-night feeds to protect the mother's (and baby's) circadian rhythms, complemented by subtle CSS 3D wireframe animations.
* **Bilingual:** Full support for EN and RU interfaces.

## 🛠 Tech Stack
* **Frontend:** React 18, Tailwind CSS, Babel (Standalone)
* **Backend:** PHP (Session Management)
* **Design:** Full Responsive SaaS Layout (CSS Grid/Flexbox)

## 🏃‍♂️ How to Run
This is a lightweight, server-rendered demo. No Node.js build steps required.
1. Clone the repository.
2. Place `index.php` and `app.jsx` in any PHP-supported web server directory (Apache, Nginx, XAMPP, etc.).
3. Open the browser and navigate to the directory.
